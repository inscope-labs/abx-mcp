# Process Report: Fix Stage 0 Build Compilation

- **Timestamp**: 2026-07-13T23-50-41Z
- **Task Slug**: fix-stage0-build

## 1. What was asked
The project is configured for a multi-stage compilation flow (stages 0-8). In stage 0, the build failed due to Kotlin compilation task (`compileDebugKotlin`) attempting to compile files inside the main source set (`app/src/main/java/com/inscopelabs/abxmcp/EnrollmentScreen.kt` and others), which produced hundreds of `Unresolved reference` errors because standard UI libraries (e.g., Jetpack Compose, Material 3, ZXing) are not applied or accessible in stage 0's classpath.

The goal was to fix all stage 0 compilation issues without violating scope discipline or breaking other configurations.

## 2. What was changed

### Modified Files

#### `app/build.gradle.kts`
Surgically updated the `KotlinCompile` task configuration block to dynamically exclude all files residing in `src/main/java` during compilation when `stage == 0`.

**Diff:**
```kotlin
tasks.withType<org.jetbrains.kotlin.gradle.tasks.KotlinCompile>().configureEach {
  if (stage == 0) {
    exclude { element ->
      element.file.absolutePath.contains("src/main/java")
    }
  }
}
```

This ensures that only stage 0 source files are compiled when the `stage` property is set to 0, completely isolating stage 0 build from any downstream/main source files without relying on complex, fragile AGP source set overrides.
