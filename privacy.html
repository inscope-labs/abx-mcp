ABC Server

Auto Bridge Context Server (ABX-MCP)

Last Updated: July 10, 2026

1. Introduction

Welcome to **ABC Server** (internally referred to as ABX-MCP), a native Android application designed to act as a secure, hardware-backed, session-gated, and capability-scoped bridge between Model Context Protocol (MCP) clients and the local Android filesystem.

Your privacy and security are the foundational pillars of our software design. This Privacy Policy outlines precisely how we handle access to your device, how cryptographic keys are managed, and how we ensure your data never leaves your control.

Core Invariant: ABC Server is an offline-first, local-only application. We do not operate secondary backend servers, track your behavior, or collect and transmit any data to third parties.

2. No Collection of Personal Data

We do not collect, store, upload, or transmit any of your personal information, telemetry, usage statistics, or device logs. All data processing remains strictly local to your physical Android device. Specifically:

  • Filesystem Data: Any access, read, or write operation performed on files and directories on your device occurs entirely locally. Files are parsed by the on-device MCP executor and transmitted directly and exclusively to the specific MCP client you have securely paired with.
  • Cryptographic Keys: Cryptographic material and hardware-backed attestation keys are securely generated inside the Android KeyStore System (using TEE/StrongBox where available). These keys are strictly non-exportable and never leave the device.
  • Session Tokens: Temporary tokens used to gate and authorize active MCP client sessions are stored transiently in local, on-device memory and automatically expire when a session terminates.

3. Secure Gating & User Consent

ABC Server does not allow silent, remote, or automatic background connections. The application enforces a strict security posture requiring direct, physical human gesture approval to initiate or extend any bridge connection:

  • Connections must be authorized via explicit interaction with the local on-device UI (such as physical button presses).
  • Active connections are clearly surfaced in the system drawer using a persistent Foreground Service notification, so you always know when a bridge is live.
  • The bridge enforces strict time-to-live (TTL) countdowns, shutting down communication interfaces automatically upon expiry unless manually renewed.

4. Permissions Used & Purpose

The application requests specific standard Android permissions to fulfill its technical role:

  • INTERNET: Required to establish local websocket or secure tunnel network socket connections to paired MCP client tools. No arbitrary outbound surfing or tracking occurs under this permission.
  • POST_NOTIFICATIONS: Required on Android 13+ (API 33) to display the live foreground status notification of active server sessions, ensuring visual transparency.
  • Storage Access / SAF (Storage Access Framework): Requested dynamically during runtime when you configure directory allowed roots to authorize access to specific folders. The application can only interact with files in folders you have explicitly designated.

5. Audit Logging

To provide security transparency and diagnostics, the application generates append-only, hash-chained, and local-key-signed audit logs tracking security events (such as policy rejections, unauthorized access attempts, or session violations). These logs are stored strictly on-device, are not shared with any external parties, and are under your complete control.
