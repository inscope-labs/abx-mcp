Data Deletion

Instructions for ABC Server (ABX-MCP)

Last Updated: July 10, 2026

1. Our Data Philosophy

**ABC Server** (ABX-MCP) is designed from the ground up as a fully local, on-device application.

We do not collect, store, or transmit any user data to external servers or cloud environments. There is no account system, no cloud backend, and no remote database holding your personal information. Consequently, you are in full and exclusive control of your data at all times.

Because no data ever leaves your device, there is no separate deletion request process, waiting period, or backend to contact. Deleting your local data removes it immediately and permanently.

2. Steps to Delete Your Local Data

You have direct control over all state and configuration files generated by the application. You can completely wipe all local data using either of the following standard Android methods:

  1. Option 1: Clear Application Storage

    Wipe all active configurations, logs, and cryptographic parameters without uninstalling the application:

    • Open your Android device's Settings app.
    • Navigate to Apps (or Apps & Notifications) → ABC Server.
    • Tap Storage (or Storage & Cache).
    • Tap Clear Data (or Clear Storage). This immediately and irreversibly wipes all locally stored session state, audit logs, and cryptographic keys generated by the application.
  2. Option 2: Uninstall the Application

    Permanently remove the application and all of its associated sandboxed files:

    • Press and hold the ABC Server app icon on your home screen or app drawer.
    • Drag it to the Uninstall region or tap Uninstall.
    • Confirm the uninstallation. This immediately and permanently deletes all associated local data, including the Android Keystore-backed cryptographic keys tied to the application.

3. Local Data Types and Retention

For complete technical transparency, here is a detailed breakdown of the data types processed locally and how they are retained:

Data Type Where Stored Retention Period & Deletion Behavior
Session Tokens Device Volatile Memory (RAM) Held transiently during active communication sessions. Automatically cleared and forgotten on session termination.
Audit Logs On-device Protected Storage Stored locally for security transparency. Deleted instantly upon clearing app data or uninstalling the app. No remote backups or copies are kept.
Cryptographic Keys Android Keystore System (TEE/StrongBox) Non-exportable cryptographic hardware entries. Instantly and permanently deleted upon app uninstallation or clearing storage.

Once deleted via the local settings or uninstallation, **no data is retained anywhere**—there is no recovery grace period, off-site backup, or server copy.
